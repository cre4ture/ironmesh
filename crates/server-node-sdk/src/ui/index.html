Node bootstrap

Issues a node bootstrap artifact via POST /auth/node-bootstraps/issue. Use it with IRONMESH_NODE_BOOTSTRAP_FILE on the joining node.

Lifecycle note: For long-lived production nodes, prefer a node enrollment package. A plain node bootstrap plus direct cert/CA file paths is mainly for development/testing, externally managed certificates, or other short-lived/manual setups.

sign in or provide an admin token override, then click the button to issue a node bootstrap artifact

Node enrollment packages can generate internal mTLS material and, when public TLS paths are included, public HTTPS certificates too. They require IRONMESH_INTERNAL_TLS_CA_KEY for internal certs and IRONMESH_PUBLIC_TLS_CA_KEY for dedicated public certs. If both listener tiers use the same CA, the internal CA key is reused automatically. Issued material now includes validity and renewal metadata so you can rotate before expiry, and the bootstrap carries an enrollment issuer URL for node-side automatic renewal. This enrollment path is the supported production lifecycle path for node trust and certificate management.

click the enrollment button to issue a bootstrap plus generated TLS PEM material

Issue Enrollment From Join Request

Imports a joining node's request blob and issues a node enrollment package via POST /auth/node-join-requests/issue-enrollment. This is the cluster-side companion to the first-run bootstrap join flow.

paste a node join request and click the button to issue an enrollment package for that node

Node enrollment renewal

Reissues TLS material for an existing enrollment via POST /auth/node-enrollments/renew. Paste the current node enrollment JSON to preserve the same bootstrap identity while rotating certs.

paste an existing node enrollment package and click the button to reissue TLS material

Node certificate status

Fetches the current lifecycle state for configured public and internal certificates via GET /auth/node-certificates/status. Enrollment-based nodes can automatically refresh the package on disk when IRONMESH_NODE_ENROLLMENT_RENEWAL_ADMIN_TOKEN is configured; use IRONMESH_NODE_ENROLLMENT_AUTO_RENEW_ENABLED and IRONMESH_NODE_ENROLLMENT_RENEWAL_CHECK_SECS to control that behavior. Direct env/file CA wiring is primarily intended for development/testing or short-lived manual setups and may require a restart after trust-root changes.

click the button to inspect current certificate expiry and renewal state

Managed signer backup

Exports or imports the managed signer CA used by the zero-touch setup path. Export returns an encrypted JSON backup via POST /auth/managed-signer/backup/export. Import persists the backup on this node via POST /auth/managed-signer/backup/import and requires a restart before the imported signer material is active for issuance.

provide a backup passphrase and click the button to export an encrypted managed signer backup

paste an encrypted managed signer backup and click the button to import it on this node

Available routes

  • GET / - info page
  • GET /logs?limit=<n> - latest server log lines
  • GET /health - node health JSON
  • GET /snapshots - snapshot metadata
  • GET /store/index - directory index with cached media metadata when available
  • PUT /store/{key} - store object bytes
  • POST /store-chunks/upload - upload one deduplicated content chunk (hash returned)
  • POST /store/{key}?complete - finalize object from uploaded chunk refs
  • GET /store/{key} - fetch object bytes from latest state
  • GET /media/thumbnail?key=<path> - cached on-node image thumbnail for gallery views
  • GET /store/{key}?snapshot=<id> - fetch object from snapshot state
  • GET /store/{key}?version=<version_id> - fetch object by specific version
  • GET /store/{key}?read_mode=preferred|confirmed_only|provisional_allowed - read latest via explicit consistency mode
  • GET /versions/{key} - list version DAG metadata
  • POST /versions/{key}/commit/{version_id} - commit version (quorum policy aware)
  • POST /versions/{key}/confirm/{version_id} - compatibility alias for commit endpoint
  • GET /cluster/status - cluster summary
  • GET /cluster/nodes - known node list
  • PUT /cluster/nodes/{node_id} - register/update node metadata using nested reachability/capabilities
  • DELETE /cluster/nodes/{node_id} - remove node from cluster membership
  • POST /cluster/nodes/{node_id}/heartbeat - refresh node liveness
  • GET /cluster/placement/{key} - deterministic placement decision
  • GET /auth/admin/session - inspect current admin session/login status
  • POST /auth/admin/login - create a local admin session from the regular zero-touch admin password
  • POST /auth/admin/logout - clear the current local admin session cookie
  • POST /auth/bootstrap-bundles/issue - issue a bootstrap JSON bundle for client enrollment (admin session or token required)
  • GET /auth/client-credentials - inspect enrolled client credential metadata including fingerprints and revocation state (admin session or token required)
  • DELETE /auth/client-credentials/{device_id}?reason=<text> - revoke an enrolled client credential and optionally record a revocation reason (admin session or token required)
  • POST /auth/node-bootstraps/issue - issue a node bootstrap JSON artifact for provisioning a server node (admin session or token required)
  • POST /auth/node-enrollments/issue - issue a node enrollment package with generated internal mTLS material and optional public HTTPS certificates (admin session or token required, CA key material required)
  • POST /auth/managed-signer/backup/export - export an encrypted managed signer backup for recovery or signer transfer (admin session or token required, signer CA material required)
  • POST /auth/managed-signer/backup/import - import an encrypted managed signer backup onto this node; restart required before it becomes the active signer (admin session or token required)
  • POST /auth/node-join-requests/issue-enrollment - issue a node enrollment package from a joining node's request blob (admin session or token required, CA key material required)
  • POST /auth/node-enrollments/renew - renew an existing node enrollment package with fresh TLS material for the same node identity (admin session or token required, CA key material required)
  • GET /auth/node-certificates/status - inspect current public/internal certificate expiry and renewal state (admin session or token required)
  • GET /cluster/replication/plan - current replication gaps/overages
  • POST /cluster/replication/audit - manual audit trigger
  • POST /cluster/replication/repair - execute one-pass replica repair for missing placements
  • GET /cluster/reconcile/export/provisional - export local provisional metadata for rejoin sync
  • POST /cluster/reconcile/{node_id} - import provisional commits from a peer node
  • POST /maintenance/cleanup?retention_secs=<n>&dry_run=true|false - retention-safe orphan cleanup for manifests/chunks
